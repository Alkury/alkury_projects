# bot_custom_emoji

Telegram-бот на [aiogram](https://docs.aiogram.dev/) для просмотра **идентификаторов кастомных эмоджи** (`custom_emoji_id`), которые Telegram передаёт в сообщениях с наборами эмоджи / Premium.

## Возможности

- **`/start`** — краткая инструкция и контакт разработчика.
- **`/ex`** — пример использования кастомных эмоджи в HTML-сообщениях (тег `<tg-emoji>`).
- Любое другое текстовое сообщение (или подпись к медиа): бот ищет сущности типа `custom_emoji` и отвечает списком: превью эмоджи и соответствующий **`custom_emoji_id`**.

Обычные Unicode-эмоджи **не** имеют `custom_emoji_id` — в ответе API для них нет сущности `custom_emoji`, бот сообщит, что кастомных эмоджи не найдено.

## Требования

- Python 3.10+
- Зависимости из `requirements.txt`

## Установка и запуск

```bash
cd bot_custom_emoji
pip install -r requirements.txt
```

Токен бота от [@BotFather](https://t.me/BotFather) можно задать одним из способов:

1. В файле `config.py`: `BOT_TOKEN = "ваш_токен"`
2. Переменная окружения: `export BOT_TOKEN='ваш_токен'`

Запуск:

```bash
python main.py
```

## Структура проекта

| Файл | Назначение |

| `main.py` | Обработчики команд и сообщений, разбор `MessageEntity` |

## Как это устроено

Telegram Bot API помечает кастомные эмоджи сущностями с типом `custom_emoji` и полем `custom_emoji_id`. Смещения `offset` и `length` в сущностях задаются в **UTF-16 code units**; в коде для вырезания фрагмента текста используется функция, совместимая с этим правилом.
